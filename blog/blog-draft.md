# blog-draft

> 11dy.tistory.com 톤·구조에 맞춰 **현재 세션의 작업 내용**을 기술 블로그 초안 마크다운으로 정리하는 스킬.

- **버전**: 0.2.0
- **대상 블로그**: https://11dy.tistory.com (DY_DEV)
- **출력 위치**: `~/dev/log_for_blog/`
- **출력 포맷**: 플랫폼 중립 단일 마크다운 (frontmatter 없음)

## 트리거

- 슬래시 명령: `/blog`
- 자연어: "블로그 초안 써줘", "이거 글로 정리해줘", "이거 트러블슈팅 글로 정리해줘", "블로그 글로 만들어줘"

## 입력

**현재 세션의 대화 컨텍스트**가 입력 소스다. 사용자가 진행한 작업 흐름:
- 무슨 작업을 하다가
- 어떤 문제를 발견했고
- 어떤 가설/시도를 거쳤고
- 어떻게 해결했는지

외부 work-log 파일이나 다른 디렉토리는 참조하지 않는다.

부족한 정보는 한 번에 모아서 사용자에게 묻는다.
- 개발환경 정보(스택, 버전)
- 회고에 담을 한 줄 코멘트
- 참고 링크 (있다면)

## 출력

- **경로**: `~/dev/log_for_blog/{topic}_{YYYY-MM-DD}.md`
- **파일명 규칙**:
  - `{topic}` — 세션의 핵심 주제 slug (영문/한글 OK, 공백·특수문자 → 하이픈, 30자 이내 권장)
  - `{YYYY-MM-DD}` — 작성일
  - 예: `fluentbit-kube-tag-prefix_2026-05-03.md`, `spring-security-filterchain_2026-05-03.md`
  - 동일 파일명 존재 시 끝에 `-2`, `-3` 붙여 충돌 방지
- **포맷**: frontmatter 없는 단일 마크다운. Tistory/Jekyll 분기 출력하지 않는다.

## 글 종류 판별

| 종류 | 신호 | 제목 패턴 |
|------|------|-----------|
| 트러블슈팅 | 에러 메시지, 예상치 못한 동작, "왜 안 되지" | `[트러블슈팅] {기술A} + {기술B} {증상}` |
| 학습-개선 | 성능 측정, 신규 도입, 비교/선택 | `{기술}을 이용한 {목표/성과}` |

판별 어려우면 사용자에게 한 번 묻는다.

## 글 구조 — 트러블슈팅

```
# [트러블슈팅] {제목}

{도입부 2~3문장: 어떤 작업 중이었고, 어떤 문제가 발생했고, 왜 기록을 남기는지}

#### 개발환경
- {스택1} {버전}
- {스택2} {버전}

### 문제 상황
{증상 묘사 + 에러 메시지/로그 인용}

[이미지: 문제 발생 로그]

### 예상되는 원인
- {가설1}
  - {세부원인1}
- {가설2}

### {가설1 검증}
{검증 과정과 결과}

### 해결
{실제 원인} → {적용한 수정}

[이미지: 수정 후 화면]

{회고 한 줄. 가벼운 자조 + 이모지 1개. 예: "사소한 문제로 많은 시간을 소모한 것 같습니다. {분야} 공부가 더 필요한 것 같네요...🤦‍♂️"}

읽어주셔서 감사합니다!!

참고
- {URL1}
- {URL2}

> 카테고리: Spring
> 태그: Spring Boot, Redis, UnsatisfiedDependencyException
```

## 글 구조 — 학습-개선

```
# {기술}을 이용한 {목표/성과}

{도입부 2~3문장}

개발환경
- {스택1} {버전}

## {측정/문제 정의}
{측정 방법, 결과 데이터}

## {왜 이 방법인가}
- 첫 번째, **{특징1}**합니다. {부연}
- 두 번째, **{특징2}**합니다. {부연}

## {기술} 설정 과정
#### {파일명}
```{언어}
{코드}
```

## 적용하기
{어노테이션/메서드 단위 적용}

## {결과 비교}
{Before/After}

참고
- {URL1}

> 카테고리: Spring
> 태그: ...
```

## 톤 규칙 (HARD RULES)

1. **존댓말 일관**. "~합니다", "~었습니다". 반말/명령형 금지.
2. **1인칭 "저는"** 사용. "필자", "본 글에서는" 격식체 금지.
3. **인사말 금지**. "안녕하세요", "여러분", "이번 포스팅에서는~" 금지.
4. **메타 문장 금지**. "지금부터 설명하겠습니다" 같은 안내 문장 금지. 흐름은 소제목으로만.
5. **이모지는 회고 줄에만 1개**. 본문 중간 이모지 금지. 허용: 🤦‍♂️ 😂 🚨
6. **굵게는 핵심 한 문장에만**. 섹션당 1~2개 이내.
7. **참고 링크는 raw URL**. `[제목](url)` 형식 X. (Tistory 자동 임베딩 카드 활용)
8. **코드블록 직전 파일명은 `#### 파일명`** 라벨링.
9. **에러 메시지는 인용 블록(`>`)** 으로.
10. **회고는 한 줄**. 두 줄 이상 늘어지면 11dy 톤이 깨진다.

## 워크플로우

1. 현재 세션 컨텍스트 스캔 (작업 흐름 파악)
2. 글 종류 판별 (트러블슈팅 / 학습-개선)
3. 부족 정보 한 번에 질문
4. 템플릿 채우기
5. 톤 규칙 self-check
6. `~/dev/log_for_blog/{topic}_{YYYY-MM-DD}.md` 저장
7. 사용자에게 절대경로 + 한두 줄 요약 보고

## Self-check 체크리스트

- [ ] 인사말/메타문장 없음
- [ ] 존댓말 일관
- [ ] 굵은 글씨 섹션당 1~2개 이내
- [ ] 회고 한 줄 + 이모지 1개
- [ ] 참고 링크 raw URL
- [ ] 코드블록 위 `#### 파일명` 라벨
- [ ] 에러 메시지 인용 블록 처리
- [ ] 태그에 정확한 에러 메시지 포함 (트러블슈팅)
- [ ] 파일명이 `{topic}_{YYYY-MM-DD}.md` 형식
- [ ] `~/dev/log_for_blog/`에 저장됨

## 절대 하지 말 것

- LLM 흔적 ("위에서 살펴본 것처럼", "결론적으로 말씀드리면")
- 일반론 단락 ("Spring Security는 강력한 보안 프레임워크입니다" 같은 교과서 문장)
- **세션에 없는 디버깅/측정/에러 메시지 창작 금지**
- 참고 URL 만들어내기 — 세션에서 사용자가 명시·인용한 것만
- Tistory/Jekyll용 frontmatter나 분기 출력
